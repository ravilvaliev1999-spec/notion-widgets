# notion-widgets

Виджеты для Notion. Актуальный виджет «Файлы задачи» — **taskfileswidgetv18.html**
(эта же ссылка вшита в шаблон «Новая задача»; менять её не нужно).

## Как это работает теперь

**1. Виджет открывается мгновенно — всегда.**
Никакого экрана «Привязываю виджет к этой задаче…». Блоки Drive / Docs / Sheets / Slides
рисуются сразу из localStorage, до события `load` виджет не делает ни одного сетевого
запроса (Notion крутит спиннер, пока iframe не загрузится, поэтому любые запросы к
`script.google.com` на старте растягивали открытие на секунды).

**2. Новая задача = пустой виджет, привязка происходит сама и в фоне.**
Пока в задаче нет файлов, определять страницу незачем. Как только вы сохраняете первый
файл, виджет получает временный ключ и тихо просит Apps Script пометить embed-блок
меткой `#task=<id страницы>` через Notion API. Apps Script возвращает id страницы, и
виджет переключается на постоянный ключ сразу, не дожидаясь перезагрузки блока — файлы
не теряются и не «переезжают» в другие задачи: у каждой задачи свой ключ.

**3. Скачивание работает из песочницы Notion.**
Notion вставляет виджеты в iframe без разрешения `allow-downloads`, поэтому обычный
клик по ссылке скачивания браузер молча блокирует («скачивание началось», а файла нет).
Теперь виджет открывает крошечное окно-«курьер»: оно живёт вне песочницы
(`allow-popups-to-escape-sandbox`), запускает скачивание и закрывается само (~1 сек).
В десктопном приложении Notion файл скачивается по прямой ссылке Google Drive через
браузер. При открытии виджета напрямую (не в iframe) — обычное скачивание без окон.

**4. Файл с компьютера добавляется мгновенно, Excel скачивается в оригинале.**
Карточка появляется сразу, загрузка в Drive идёт в фоне; всё это время файл уже можно
скачивать — байты хранятся локально (IndexedDB, без лимита localStorage). Если загрузка
в Drive сорвалась (закрыли страницу, Apps Script спал) — виджет дозагрузит файл при
следующем открытии. Форматы не конвертируются: xlsx скачивается байт-в-байт.

## Установка / обновление

1. **Виджет.** GitHub Pages уже включён; embed-ссылка для блоков:
   `https://ravilvaliev1999-spec.github.io/notion-widgets/taskfileswidgetv18.html`
   У привязанных блоков сохраняйте хвост `#task=…`.

2. **Apps Script (обязательно).** Откройте проект на [script.google.com](https://script.google.com),
   замените код содержимым [`apps-script/Code.gs`](apps-script/Code.gs) и **переразверните**:
   Deploy → Manage deployments → ✏ → Version: **New version** → Deploy
   (Web app, Execute as: Me, Who has access: Anyone).
   Без новой версии рабочий URL продолжит отдавать старый код — у виджета не будет
   мгновенной привязки (команда `tag` со списком страниц) и скачивания (`download`).

3. **Автопривязка.** Project Settings → Script properties → `NOTION_TOKEN` = токен
   интеграции Notion; страницы задач должны быть подключены к интеграции (Connections).

`SECRET` в Code.gs должен совпадать с `WIDGET_KEY` в HTML виджета (уже совпадает).

## Шаблон задачи, synced-блоки и «маяки»

Рабочий процесс: эталон виджета лежит в synced-блоке на странице «Системные
виджеты», его sync-копия — в шаблоне «Новая задача». **Synced-блок никогда не
меняется и не удаляется** — ни в шаблоне, ни в задачах.

Так как у synced-блока один URL на все задачи, виджет сам по себе не знает,
на какой он странице. Эту роль выполняет «маяк» — крошечный невидимый
(прозрачный) embed-блок в конце страницы задачи с адресом
`taskfileswidgetv18.html?beacon=1#task=<id страницы>`. Маяк ничего не рисует и
только сообщает виджету ключ задачи через postMessage между фреймами страницы.
Привязка через маяк мгновенная и работает даже без Apps Script.

Маяк создаётся автоматически: при первом открытии новой задачи Apps Script
находит её напрямую в базе «4. Задачи» и добавляет блок-маяк в конец страницы.
Там же чинятся маяки с чужой меткой (такое бывает при дублировании задачи).
Страницы вне базы задач не затрагиваются никогда. Если маяк случайно удалить,
он появится снова при следующем открытии задачи.

Если база задач другая — задайте её id в Script properties: `TASKS_DB_ID`.

## Ограничения

- Загрузка файла с компьютера через Apps Script — до 15 МБ; скачивание через скрипт —
  до 30 МБ (больше — открывайте в Drive по кнопке ↗).
- Список файлов хранится в localStorage браузера и не синхронизируется между
  устройствами; сами файлы лежат в Drive («Notion — файлы задач/<задача>»), поэтому
  на другом устройстве скачивание тоже работает — байты приедут из Drive.
- В веб-версии Notion для скачивания должны быть разрешены всплывающие окна
  (по умолчанию разрешены — окно открывается по явному клику).
